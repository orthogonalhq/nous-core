# AGENTS

This repo follows the Agent Engineering Workflow SOP.
Source of truth is the repo docs. Manual approval at every gate.

## Read first
- docs/sop/agent-engineering-workflow.md

## Required doc paths
docs/
  overview/
  architecture/sds/
  architecture/adr/
  roadmap/
  specs/business/
  specs/implementation/
  operations/
  engineering/

## Checkpoints (stop if missing)
1) Intake
- Business Spec exists: docs/specs/business/...
- Acceptance criteria included
- Approved by human

2) Design
- SDS exists: docs/architecture/sds/...
- ADRs created if durable decisions
- Approved by human

3) Decompose
- Roadmap subphase exists: docs/roadmap/...
- Definition of Done included
- Approved by human

4) Plan
- Implementation Spec exists: docs/specs/implementation/...
- Includes tasks, tests, validation, rollback
- Approved by human

5) Implement + Verify
- PR includes required links and checklist
- CI green (lint/typecheck/tests)
- Approved by human

6) Ratify
- Docs updated to match reality (same PR or follow-up PR immediately)
- ADR updated with PR link (if ADR exists)

## ADR rule
Write an ADR when the decision is durable, cross-cutting, hard to reverse, or changes contracts/ops posture.
