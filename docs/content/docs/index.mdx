# Jarvis-OSS

Jarvis-OSS is a local-first agent operating system.

It is designed to provide structured orchestration, explicit memory control, model routing, and secure tool execution — without depending on a single model provider or centralized service.

Jarvis is not a chatbot wrapper.

It is a control plane for intelligent systems.

---

## Intro

Recent agent systems focus on autonomy but often blur execution boundaries, memory persistence, and model control. Jarvis approaches the problem differently.

The system is built around a Prefrontal Cortex model — a dedicated orchestration layer responsible for routing, reflection, memory gating, and tool authorization.

Models advise.

The PFC decides.

Jarvis is designed to:

- Support local and remote models
- Operate offline when possible
- Enforce explicit memory writes
- Prevent ambient authority
- Isolate projects
- Prepare for a curated skills ecosystem

The goal is clarity, not chaos.

---

## Objective

The objective of Jarvis-OSS is to build a stable, extensible, local-first agent runtime that can serve as the foundation for:

- Personal agents
- Project-scoped AI systems
- Tool-driven automation
- Secure skill distribution
- Enterprise-grade policy enforcement
- A curated skills marketplace

Jarvis prioritizes:

- Deterministic architecture
- Explicit control over execution
- Transparent memory systems
- Role-based model routing
- Capability-gated tool access
- Cross-platform compatibility

This is infrastructure, not a prompt experiment.

---

## Roadmap

Jarvis development is structured in phases.

### Phase 1 — Core Runtime

- PFC orchestration model (tiers 0–5)
- Local and remote model support
- Role-based model routing
- Explicit STM + LTM memory model
- Project isolation
- Capability-gated tool execution
- Cross-platform CLI
- WASM-first sandbox design

This phase establishes architectural stability and delivers a beta-ready system.

→ See: `/docs/roadmap/phase-1`

---

### Phase 2 — Skill Foundation

- Formal Skill SDK
- Stronger runtime isolation
- Capability enforcement expansion
- Internal curated skill library
- agent-skills-x integration

This phase prepares the ecosystem layer without opening distribution publicly.

---

### Phase 3 — Marketplace

- Verified skill registry
- Maintainer governance
- Cryptographic signing
- Skill versioning and compatibility enforcement
- Skill manager CLI and UI

Jarvis becomes an extensible platform with a curated ecosystem.

---

Jarvis-OSS is built deliberately.

Architecture first.  
Ecosystem second.  
Scale last.
