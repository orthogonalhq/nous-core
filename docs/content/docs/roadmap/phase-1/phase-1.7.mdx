# Phase 1.7 — Installer and First-Run

**Branch:** `phase-1.7/installer-first-run`

**Size:** Medium

**Contributes to:** Milestone 1 (Installation and Setup)

**Prerequisite:** Phase 1.6 (web UI must be working — the first-run experience runs through it)

---

## Goals

Make Nous installable by anyone. When 1.7 is done, a user downloads Nous, runs the installer, and within minutes has a fully working system — local LLM running, storage initialized, configuration set, and the web UI greeting them with a first-run experience.

---

## Scope

### In Scope

- Guided installer script/application:
  - Detect operating system and architecture
  - Check system requirements (disk space, available memory)
  - Install or detect LLM provider (Ollama)
  - Download a default model (small, fast, good for general interaction)
  - Initialize document storage
  - Generate default configuration
  - Start the Nous backend and web UI
  - Open the browser to the first-run experience

- First-run web UI flow:
  - Welcome screen (what is Nous, what just happened)
  - Configuration review (model assignments, PFC tier — with sensible defaults pre-filled)
  - First interaction prompt ("Say something to Nous")
  - Confirm everything works

- Cross-platform support:
  - macOS (native terminal or .dmg if feasible)
  - Linux (shell script)
  - Windows (PowerShell script or installer)

- Error handling:
  - Clear error messages if requirements aren't met
  - Graceful fallback if Ollama is already installed
  - Recovery guidance if model download fails

### Out of Scope

- Automated updates or version management (future)
- Multiple LLM provider installation options (Ollama is the default for Phase 1)
- Docker-based installation (future consideration)
- Nous Cloud onboarding (parallel business track)

---

## Definition of Done

- [ ] Installer runs on macOS and completes without errors
- [ ] Installer runs on Linux and completes without errors
- [ ] Installer runs on Windows and completes without errors
- [ ] Ollama is installed (or detected if already present)
- [ ] Default model downloads successfully
- [ ] Document storage is initialized
- [ ] Default configuration is generated and valid
- [ ] Nous backend starts and web UI is accessible
- [ ] First-run web UI flow completes (welcome → config review → first message → confirmation)
- [ ] Clear error messages when requirements aren't met
- [ ] Graceful handling when Ollama is already installed
- [ ] Unit tests for requirement detection and config generation
- [ ] End-to-end test: clean machine → run installer → first-run completes → Nous responds to a message
