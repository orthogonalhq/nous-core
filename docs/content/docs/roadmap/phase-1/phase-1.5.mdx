# Phase 1.5 — Cortex

**Branch:** `phase-1.5/cortex-pfc-core-traces`

**Size:** Large

**Contributes to:** Milestone 3 (Can Think), Milestone 4 (Can Remember — PFC gates writes), Milestone 6 (Traces)

**Prerequisite:** Phase 1.3 (providers, router, tools, projects), Phase 1.4 (STM, MWC flow)

---

## Goals

Implement the consciousness layer. When 1.5 is done, Nous has a complete execution loop: input arrives, context is constructed, the PFC evaluates, a model is consulted, the PFC validates the output, tool execution is authorized if needed, memory write candidates are generated and PFC-gated, the response is streamed, and a trace is persisted. This is the heartbeat.

---

## Scope

### In Scope

- `self/cortex/pfc/` — basic PFC engine:
  - IPfcEngine fulfilled
  - Single-tier operation (one PFC mode for Phase 1 — tier selection is configuration, not multiple implementations)
  - Basic reflection pass (evaluate model output before returning)
  - Memory write gating (evaluate MemoryWriteCandidates → approve/deny)
  - Tool execution authorization (evaluate tool requests → approve/deny)
  - Escalation to human: basic support (log that escalation was needed — UI handling in 1.6)

- `self/cortex/core/` — central execution loop:
  - ICoreExecutor fulfilled
  - Full single-turn cycle:
    1. Input received
    2. Context constructed (STM + project config)
    3. PFC evaluates input
    4. Model consulted via router
    5. PFC validates model output
    6. Tool execution authorized if requested
    7. Memory write candidates generated
    8. PFC approves/denies memory writes
    9. Response streamed
    10. Trace persisted

- Execution traces:
  - Trace schema (what happened at each step of the cycle)
  - Trace persistence (stored per project via document store)
  - Trace includes: PFC decisions, model calls, tool authorizations, memory write approvals/denials, timing

- Integration with all prior sub-phases:
  - Router (1.3) for model invocation
  - Providers (1.3) for model execution
  - Tools (1.3) for tool execution
  - Projects (1.3) for project context
  - STM (1.4) for conversation history
  - MWC flow (1.4) for memory governance

### Out of Scope

- Multi-tier PFC implementations (Phase 1 runs one tier — tier scaling comes later)
- Multi-pass reflection (Phase 2+)
- Advanced escalation handling (Phase 5)
- Project supervision mode / long-running orchestration (Phase 5)

## Definition of Done

- [ ] PFC evaluates incoming requests
- [ ] PFC performs basic reflection on model output
- [ ] PFC gates memory write candidates (approve/deny with logged reason)
- [ ] PFC authorizes tool execution (approve/deny with logged reason)
- [ ] Core execution loop runs the full 10-step cycle
- [ ] Responses are streamed to the caller
- [ ] Execution traces capture every step of the cycle
- [ ] Traces are persisted per project
- [ ] Traces include PFC decisions, model calls, memory approvals/denials, timing
- [ ] Unit tests for PFC reflection, memory gating, and tool authorization
- [ ] Unit tests for trace generation and schema
- [ ] Integration test: full cycle from input → PFC → model → PFC validate → memory gate → response → trace
- [ ] Integration test: PFC denies a memory write and the denial is traced
