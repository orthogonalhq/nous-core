# Phase 1.3 — Subcortex Foundation

**Branch:** `phase-1.3/subcortex-providers-router-tools-projects`

**Size:** Large

**Contributes to:** Milestone 3 (Can Think), Milestone 5 (Knows Project)

**Prerequisite:** Phase 1.2 (storage and config must be working)

---

## Goals

Implement the subconscious processing layer. When 1.3 is done, Nous can route a request to a local or remote model, execute a basic tool, and maintain isolated project containers. This sub-phase delivers the hands, the routing, and the workspace — but not yet the brain (PFC) or the memory.

---

## Scope

### In Scope

- `self/subcortex/providers/` — model provider adapters:
  - Local adapter: Ollama (IModelProvider fulfilled for text models)
  - Remote adapter: OpenAI-compatible API (IModelProvider fulfilled for text models)
  - Streaming support for both providers
  - Token usage reporting
  - Stub interfaces for image/video/vision providers (deferred)

- `self/subcortex/router/` — model routing:
  - Role → provider resolution (IModelRouter fulfilled)
  - Profile-based configuration (use local for X, remote for Y)
  - Fallback rules (if primary fails, try secondary)
  - Routing decisions logged

- `self/subcortex/tools/` — basic tool execution:
  - Tool interface implementation (IToolExecutor fulfilled)
  - At least one working tool (e.g., filesystem read or echo)
  - Tool schema validation
  - Capability declaration

- `self/subcortex/projects/` — project containers:
  - Project creation and deletion (IProjectStore fulfilled)
  - Project switching (active project context)
  - Project-scoped configuration (model assignments, PFC tier)
  - Project namespace isolation (memory and traces scoped to project)
  - Project listing

- Stub interfaces for IWorkflowEngine, IArtifactStore, IScheduler, IEscalationService, ISandbox, IProjectApi (deferred)

### Out of Scope

- PFC authorization (Phase 1.5 — for now, providers and tools execute without PFC gating)
- Heterogeneous provider implementations beyond text (Phase 2+)
- Workflow execution (Phase 5)
- Artifact storage (Phase 5)
- Scheduling (Phase 5)
- Package SDK and sandbox (Phase 7)

---

## Splitting Note

This is the largest sub-phase in Phase 1. If it proves too large during implementation, split into:

- **1.3a** — Providers + Router (enables "Nous can talk to a model")
- **1.3b** — Tools + Projects (enables "Nous has a workspace")

The SOP allows this: stop and decompose rather than create a mega-PR.

---

## Definition of Done

- [ ] Ollama provider connects, sends prompts, and streams responses
- [ ] OpenAI-compatible provider connects, sends prompts, and streams responses
- [ ] Router resolves model roles to configured providers
- [ ] Router applies fallback when primary provider fails
- [ ] At least one tool executes successfully through the tool interface
- [ ] Tool validates input against declared schema
- [ ] Projects can be created, listed, switched, and deleted
- [ ] Project configuration is persisted and loaded
- [ ] Project namespace isolation enforced (storage scoped to project)
- [ ] Unit tests for each provider adapter
- [ ] Unit tests for router resolution and fallback
- [ ] Unit tests for tool execution and schema validation
- [ ] Unit tests for project CRUD and namespace isolation
- [ ] Integration test: router resolves role → provider responds with streamed text
