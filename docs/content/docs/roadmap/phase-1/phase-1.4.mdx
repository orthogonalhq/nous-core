# Phase 1.4 — Memory Foundation

**Branch:** `phase-1.4/memory-stm-mwc-flow`

**Size:** Medium

**Contributes to:** Milestone 4 (Can Remember)

**Prerequisite:** Phase 1.2 (document storage), Phase 1.3 (project namespace)

---

## Goals

Implement basic memory that persists conversations across sessions and establishes the MemoryWriteCandidate flow — the composable unit of memory governance. When 1.4 is done, Nous can store conversation history per project and the MWC pipeline exists for the PFC to gate in Phase 1.5.

---

## Scope

### In Scope

- `self/memory/stm/` — short-term memory:
  - IStmStore fulfilled
  - Conversation history read/write
  - Session persistence (conversations survive browser close and restart)
  - Project-scoped (each project has its own STM)

- `self/memory/` — MemoryWriteCandidate flow:
  - MemoryWriteCandidate creation (model proposes a memory entry)
  - Candidate structure: content, type, scope, confidence, sensitivity tags, provenance
  - Candidate pipeline: create → evaluate → approve/deny → persist (PFC evaluation is stubbed here, implemented in 1.5)
  - Approved entries persisted to document store with provenance metadata
  - Denied entries logged (visible in traces)

- Memory export: export all memory for a project as structured data
- Memory deletion: delete individual entries or all memory for a project

- Stub interfaces for ILtmStore, IDistillationEngine, IRetrievalEngine, IKnowledgeIndex, IAccessPolicy (deferred)

### Out of Scope

- Long-term memory with typed entries (Phase 2)
- STM summarization and eviction (Phase 2)
- Vector storage or semantic retrieval (Phase 2)
- Experience records and sentiment (Phase 4)
- Cross-project memory access (Phase 3)
- Memory access policies (Phase 3)

---

## Definition of Done

- [ ] MemoryWriteCandidate structure implemented with all required fields
- [ ] MWC pipeline operational (create → evaluate stub → persist)
- [ ] Approved entries persisted with provenance metadata
- [ ] Denied entries logged
- [ ] Memory export produces structured data for a project
- [ ] Memory deletion removes entries (individual and bulk)
