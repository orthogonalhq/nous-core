# Worklog

This section contains the working artifacts produced as Nous-OSS is built.

Unlike reference docs (architecture, roadmap, business), worklog artifacts accumulate over time. They capture the state at the time the work was done — the goals that guided it, the designs that shaped it, and the decisions that constrained it.

---

## Structure

### Architecture Decision Records

ADRs capture durable, cross-cutting decisions. They are numbered sequentially across the entire project and never deleted — superseded ADRs link to their replacement.

→ [ADR Index](/docs/worklog/adr)

### Phase Artifacts

Each phase and sub-phase may produce:

- **Goals** — why this work exists, what it achieves, success criteria
- **SDS** — system design spec for significant architectural changes
- **Implementation Spec** — the plan: tasks, files, tests, acceptance criteria, rollback

These live in `worklog/phase-X/phase-X.Y/`.

---

## When to Use

- **Need to understand why a decision was made?** Read the ADR.
- **Need to understand what was planned for a sub-phase?** Read the goals and implementation spec.
- **Need to understand the current system?** Read the [reference docs](/docs/architecture) instead.
